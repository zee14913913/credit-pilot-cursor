# 立即修复 Railway Web 服务配置

**问题**：Custom Start Command 被截断，导致 502 错误

---

## 🚨 快速修复（1分钟）

### 操作步骤

1. **进入 Railway**
   - 打开 Railway 项目
   - 点击 **Web 服务**

2. **进入 Settings**
   - 点击 **Settings** 标签
   - 滚动到 **Deploy** 部分

3. **修复 Custom Start Command**
   - 找到 **Custom Start Command**
   - 点击**编辑图标**（铅笔图标）
   - **完全清空**输入框（删除所有内容）
   - **留空**（不要输入任何内容）
   - 点击**保存**（对勾图标或 Update）

4. **等待部署**
   - Railway 会自动触发新部署
   - 等待 3-5 分钟

5. **验证**
   - 访问 `https://infinite-gz.net/health`
   - 应该返回正常响应（不是 502）

---

## ✅ 为什么留空？

**Dockerfile 已经正确配置**：
```dockerfile
CMD ["bash", "start.sh"]
```

**start.sh 包含完整流程**：
1. 初始化数据库
2. 启动 API 服务器

**Railway 会自动使用 Dockerfile 的 CMD**，不需要在 Custom Start Command 中重复。
